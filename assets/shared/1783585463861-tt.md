# 开发工作流（必须严格遵守）
## Phase 1 · 意图对齐（动手前）
+ 收到需求后先判断信息是否足够执行。
+ 若不足，必须：① 用 1-2 句话复述你理解的需求 ② 指出所有歧义点 ③ 给出 3-5 步的极简实施计划。
+ **必须等待用户明确确认后**，才能进入编码阶段。

## Phase 2 · 小步编码
+ 禁止一次性输出全部代码，按已确认的计划逐步推进，每一步都附带简要说明。
+ 涉及流程/状态流转的模块，先用 Mermaid 画出流程图或时序图，再给实现代码。
+ 遇到报错必须先根据日志/调用栈定位根因，禁止凭猜测直接给"疑似修复方案"。

## Phase 3 · 完成自检（缺一不可）
1. ✅ 是否完全覆盖需求？有无遗漏边界情况？
2. ✅ 代码是否可直接运行？依赖/配置/环境变量是否齐全？
3. ✅ 若修改了已有逻辑，是否评估了对关联功能的影响？

自检通过后按以下格式收尾：

> **✔**** 完成摘要**：做了什么 | 如何验证
>

## Phase 4 · 代码审查与上下文管理
+ 提交审查时主动说明"解决了什么问题"和"潜在风险点"；收到修改意见时禁止只改表面，需说明修改逻辑并确认是否影响关联代码。
+ 涉及 Git 破坏性操作（reset / force push）时必须先警告并等待确认。
+ 对话过长或涉及多个独立模块时，主动建议开启新对话。

# 编码规范
+ **修改边界**：严格按任务范围操作，禁止越界修改无关代码。
+ **删除原则**：非必要不删除原有代码，优先以注释或扩展方式处理。
+ **样式原则**：先阅读源界面现有样式，再进行修改，避免样式污染。
+ **代码格式**：所有代码必须包裹在代码块中，标注语言类型和所属文件路径，例如：

```java
// 文件：SmartCampus/base/src/.../XxxController.java
```

+ **文件缺失处理**：若需要查看的文件在库中不存在，立即中断并按以下格式说明：

```plain
⛔ 文件缺失，请提供：
- 文件相对路径：src/xxx/xxx.vue
- 用途说明：需要查看该组件的现有样式逻辑
```

+ **语言与注释**：回复统一使用中文；必要内容用标准 Markdown 展示；代码注释使用中文，关键逻辑处补充调试日志输出。

# UI 设计参考规范（小程序端通用样式）
> 以下规范来自"印章外出使用审批"业务的 index 列表页与表单页，是本项目小程序端所有新页面必须遵循的视觉基线。
>

## 1. 整体视觉风格与配色
+ **设计语言**：现代扁平高级感（Premium & Clean Minimalist UI），浅灰背景 + 纯白卡片 + 微投影 + 细腻圆角。
+ **配色**：
    - 主色：科技蓝 `#1D61F2`
    - 背景：淡灰 `#F8FAFC`
    - 标题/主文字：`#1E293B`；表单标签：`#475569`；占位/辅助文字：`#94A3B8`
    - 分割线：`#F1F5F9` / `#E2E8F0`
    - 状态色：通过 = 绿（`#059669` / `#D1FAE5` / `#10B981`）；审核中 = 橙（`#D97706` / `#FEF3C7` / `#F59E0B`）；拒绝 = 红（`#DC2626` / `#FEE2E2` / `#EF4444`）

## 2. 核心布局
+ **根容器 **`.box`：`width:100%`，`min-height:100vh`，Flex 纵向布局，背景 `#F8FAFC`。
+ **页头 **`.premiumHeader`** / **`.functionBox`：
    - 列表页：左侧竖向渐变装饰条（`linear-gradient(to bottom, #1D61F2, #00B1FF)`，`8rpx × 38rpx`，圆角 `4rpx`）+ 加粗标题（`36rpx / 900 / #1E293B`）；右侧操作按钮组。
    - 表单页：左侧业务 SVG 彩色图标（`44rpx × 44rpx`，右间距 `16rpx`）+ 加粗标题（`36rpx / 800`）。
+ **卡片 **`.premiumCard`：纯白背景，圆角 `24-28rpx`，内边距 `32rpx`，底部外边距 `24rpx`，投影 `0 4rpx 16rpx rgba(0,0,0,0.04)`；卡片头部左侧亮蓝装饰块（`8rpx × 32rpx`，圆角 `4rpx`）+ 标题（`30rpx / 800 / #1E293B`）。

## 3. 组件级样式
+ **选项卡**：
    - 双栏/药丸切换：外层容器 `#E2E8F0`，圆角 `20rpx`，内边距 `6rpx`；激活项白底 + 轻投影，文字 `#1D61F2 / 700`；未激活 `#64748B`。
    - 下划线切换：激活项 `#1D61F2 / 800`，底部渐变指示条（`linear-gradient(90deg, #1D61F2, #00B1FF)`，`80rpx × 6rpx`，居中）。
+ **表单行 **`.formRow`：Flex 两端对齐 + 垂直居中，底边框 `1rpx solid #F1F5F9`（末行无边框）。
    - 标签 `.formLabel`：左对齐，`28rpx / 600 / #475569`，最小宽 `160rpx`。
    - 值 `.formInput / .formSelectValue / .formValue`：右对齐，`28rpx / #1E293B`；只读/禁用态 `#94A3B8`；选择项带灰色小三角 `▾`。
+ **多行输入 **`.formTextarea`：`width:100%`，`height:120rpx`，背景 `#F8FAFC`，边框 `1rpx solid #E2E8F0`，圆角 `16rpx`，内边距 `20rpx`，`28rpx`；禁用态背景 `#F1F5F9`。
+ **列表状态条 **`.statusBar`：卡片左侧贴边竖条，`8rpx` 宽，颜色随通过/审核中/已拒绝状态变化。

## 4. 按钮与交互反馈
+ **主按钮**：渐变背景（如 `linear-gradient(135deg, #1D61F2, #0A4ECB)`），蓝色投影 `0 8rpx 20rpx rgba(29,97,242,0.15)`，圆角 `20rpx`。
+ **次要/上传按钮**：纯色 `#1D61F2`，圆角 `12rpx`，内边距较小，`26rpx / 600`。
+ **触按反馈**：点击时 `transform: scale(0.96~0.98)`，可配合 `opacity:0.95`。
